# The software

The software is written in Typescript in its all entirety and consists in:

  • a Web App
  • some Rest APIs
  • an SDK

# The Web App

The Web App is written in Vue 2 (opens new window) and Typescript (opens new window). The framework is one of the four most popular frontend frameworks (namely: React, Vue, Angular, Svelte) and was chosen both because it was the better known by the developer and because its design is more elegant than React's one.

Almost all the style was written by hand and no UI framework has been used. Most of the style relies on flexbox css rules, with some use of the grid ones. This choice was done because custom style was asked by the company, with a sort of design-mockup created by a designer.

The only third-party libraries used for the style are:

  • pdfjs (opens new window): It is used to display preview (the first page) of PDF files, before they are downloaded
  • progressbar.js (opens new window): It is used for the "piadine", the semicircles with Intertek's score in the personal page of a ghostmaker
  • v-caledar (opens new window): It is used for the events calendar
  • vue-advanced-cropper (opens new window): It is used for the various image croppers that appear when uploading an image
  • vue-aos (opens new window): It is used for the animation on-scroll effects
  • vuebar (opens new window): It was used for the chat, is no more needed in the HModa version

Other non-ui libraries used are:

  • VueRouter (opens new window): The default router of Vue
  • Vuex (opens new window): The default store of Vue
  • Vue 18n (opens new window): A internationalization library for Vue
  • Vue meta (opens new window): A meta tags library for Vue, for a better SEO
  • Vue gtag (opens new window): A Google Analytics library for Vue
  • translate (opens new window): A translation library for Javascript, used to translate pages with user-generated content

# A note on the Vue version

Vue.js 2 has been used because Vue.js 3 was not out yet when the project started. Because it does not depend on a lot of third-party libraries, it should be possible to pass to the newer version, but doing it would mean rewrite a lot of code.

# The shared components

A folder shared in the Vue.js code contains components that, with some adjustments, could be really become a "Ghostmakers" UI framework, that could be reused in other Vue.js projects (web projects in general if transformed with webcomponents).

The site folder contains other interesting components, too. Some of them could be added to an Ui framework while others are too specific.
